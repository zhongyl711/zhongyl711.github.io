# 工具方法

ynet-utils库，提供了账号、数组、日期、id、身份证、金额、字符串的工具函数

# API

# 账号 - accountUtil

方法 参数 返回值
mask: 为账号增加掩码 value: string 账号
maskType: 0 保留前6位及后4位中间用*表示，1 四个*加后四位，默认0		 转换后带掩码的账号，类型string
last: 返回账号后几位 value: string 账号
size?: number 保留后几位默认为4		 截取后的账号，类型string
format: 格式化账号4位一个空格 value: string 账号 四位一个空格的账号，类型string
parse: 去除账号空格 value: string 账号 去除所有空格的账号，类型string

# 数组 - arrayUtil

方法 参数 返回值
without: 去除数组中符合过滤条件的数据 arr: Array<T> 数组
filter: ((item: T) => boolean ) | T 返回布尔值的函数或者数组中的某一项		 过滤后的数组

# id - idsUtil

方法 参数 返回值
guid: 获取唯一标识符 无 guid_ 开头拼接随机数，类型string
random: 获取随机字符串 length: number = 32,指定有多少位字符串，默认32 string

# 身份证 - idcardUtil

方法 参数 返回值
mask: 身份证掩盖，保留前6位 后4位 value: string 身份证号 string
validate: 校验格式是否是身份证 value: string 身份证号 布尔值，true表示身份证格式正常，fasle表示身份证格式错误
format: 身份证号格式化 value: string 身份证号 以3 4 4进行格式化的值

# 金额 - moneyUtil

方法 参数 返回值
format: 货币格式化 value: string 金额
digits: number 保留小数后多少位，默认2
autoFill: boolean = true 是否自动补位 默认自动补位
noRoundFlag: boolean = false 是否不四舍五入 默认四舍五入
addComma: boolean = true 是否添加逗号 默认添加		 格式化后的金额，类型string
parse: 将格式化后的金额格式化回不带“，”的字符串 value: string 金额 string
toChinese: 大写货币金额 value: string 金额 string

# 字符串 - stringUtil

方法 参数 返回值
isStrEmpty: 判断字符串是否为空 value: any 布尔值 true表示为空，false表示不为空
charLength: 字符串长度（汉字2字符） value: string number

# 数字 - numberUtil

方法 参数 返回值
plus: 加法 value: number 第一个加数 number 第二个加数 number 进行加法计算后的数值，科学计数法
minus: 减法 value: number 传入的第一个数字（被减数） number 传入的第二个数字（减数） number 进行减法计算后的数值，科学计数法
times: 乘法 value: number 第一个因数 number第二个因数 number 进行乘法计算后的数值，科学计数法
divide: 除法 value: number 传入的第一个数字（被除数） number传入的第二个不为0的数字（除数） number 进行除法计算后的数值，科学计数法
round: 四舍五入 value: number 传入的数字 number 保留小数的位数 number 进行四舍五入计算后的数值，科学计数法
decimalComplementZero: 小数位补足0 value: string 传入的数字 number 补足的位数 string 小数位补足之后的值，科学计数法

# 如何使用

# 1.项目package.json文件中导入@ynet/utils库

"dependencies": {     
    "@ynet/utils": "1.0.0"
}

# 2.在需要使用工具库函数的文件中导入对应的工具库,然后调用需要的方法即可

import { accountUtil, dateUtil } from "@ynet/utils";
accountUtil.mask("888888888888888"); // "888888 ******** 8888"
dateUtil.format("20200606", "YYYYMMDD", "YYYY-MM-DD");// 2020-06-06
