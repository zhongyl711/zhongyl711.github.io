# form 表单验证

表单验证是将 form 表单组件包装到 ynet-form 组件中使用，ynet-form 进行表单提交时会对 ynet-form 中的表单元素进行相应的规则校验

ynet-form 中事件的以及属性的具体使用请参考 form 表单控件

data-role-[params]:输入框加上此属性，输入框需要满足此校验规则;属性后面的值为提示信息

# 参数说明

属性名 说明 参数说明 默认值
data-role-required 必填项 提示语句 此项为必填项
data-role-int 正整数类型 提示语句 请输入正整数类型
data-role-number 数字类型 提示语句 请填写数字类型的值
data-role-phone 11 位手机号 提示语句 请输入11位手机号
data-role-email 验证邮箱 提示语句 请输入正确的邮箱地址
data-role-idcard 验证身份证号 提示语句 请输入正确的身份证号
data-role-postcode 验证邮编 提示语句 请填写正确的邮编
data-role-account 验证账号 提示语句 账号为32位以内的数字、字母、下划线、横线组合
data-role-regex 正则校验（旧）｜正则校验提示语句（新） 正则表达式String或提示语句 -
data-role-regex-test 正则校验（新） 正则表达式String -
data-role-regex-msg 正则校验提示语句（旧） 提示语句 -
data-role-length 字符长度限制 提示语句 输入字符长度不符合要求
data-role-length-min 最小长度 Number类型 -
data-role-length-max 最大长度 Number类型 -
data-role-force 强制校验,主要用于禁用和只读状态时表单控件强制校验 - -
data-role-emoji 校验 emoji 表情 提示语句 输入内容不能包含emoji表情
data-role-special-code 特殊字符校验 提示语句 不能存在特殊字符

注意：正则校验有新老版写法，目前做了兼容处理，建议使用新版本写法

# 代码演示

1.项目package.json文件中导入@ynet/utils库

"dependencies": {     
    "@ynet/utils": "1.0.0"
}

2.项目中按需引入默认提供的校验器

import "@ynet/validate/plugins/required";
import "@ynet/validate/plugins/int";
import "@ynet/validate/plugins/number";
import "@ynet/validate/plugins/phone";
import "@ynet/validate/plugins/email";
import "@ynet/validate/plugins/idcard";
import "@ynet/validate/plugins/length";
import "@ynet/validate/plugins/regex";
import "@ynet/validate/plugins/postcode";
import "@ynet/validate/plugins/account";

3.新增校验器

import Validate from "@ynet/validate";
// 此处demo参数，只为演示 实际需求根据 data-role-[params]中params名保持一致
Validate.addRoles("demo", {
  // value为需要进行校验的值，params为 data-role-[params] 绑定的key: value
  validate: (value, params) => {
        var clientRegex = /^([0-9]|[a-z]|[A-Z]|_|-){0,32}$/;
        if (clientRegex.test(value) == false) {
            return params.demo || "账号为32位以内的数字、字母、下划线、横线组合";
        }
  }
})

注意：新增属性名字如果和提供的默认校验器同名，会覆盖默认校验器如：

import Validate from "@ynet/validate";
// 默认已经提供了 data-role-required 再次添加会覆盖默认提供的
Validate.addRoles("required", {  
  validate: (value, params) => {
    if (value === null || value === "" || value === undefined) {
      return params.required || "此项为必添项";
    }
  }
})
