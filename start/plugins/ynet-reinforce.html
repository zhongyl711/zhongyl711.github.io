# 前端防调试及加固

ynet-reinforce库使用禁止键盘事件防调试，及采用javascript-obfuscator和multimatch实现加固。

# preventDebugSetting 防调试

方法 参数 返回值
preventDebugSetting PreventDebugConfig（见下方类型）| '*'：(全部启用) 无

PreventDebugConfig：

type PreventDebugConfig = {
    F12?: Boolean;// 禁用F12
    CtrlS?: Boolean;// 禁用Ctrl+S保存
    CtrlC?: Boolean;// 禁用Ctrl+C复制
    Contextmenu?: Boolean;// 禁用右键菜单
    Debugger?: Boolean;// 开启控制台后循环debugger
};

# 如何使用

# 1.项目package.json文件中导入@ynet/request-axios库

"dependencies": {
	"@ynet/reinforce": "1.0.0"
}

# 2.在项目中应用

import preventDebugSetting from "@ynet/reinforce";
// preventDebugSetting('*') // 开启全部功能

// 开启 启动控制台 循环debugger
preventDebugSetting({
  Debugger: true
})

# 前端加固

混淆文件分为轻度，中度和重度三个等级，及可配置不加固的文件

# 如何使用

# 1.项目package.json文件中导入@ynet/request-axios库

"dependencies": {
	"@ynet/reinforce": "1.0.0"
}

# 2.在项目中应用

// 在prod.config.js中配置 查看extendPlugin配置
const { merge } = require("@ynet/cli");
const baseConfig = require("./base.config");

module.exports = merge(baseConfig, {
    compilerOption: {
        
    },
    globalDefine: {
     
    },
    extendPlugin: [
        [
        	"@ynet/reinforce",
        	{
                //加固等级 leve1：表示轻度 leve2：表示中度 leve3：表示重度
                leve: "leve2" 
                // 不传入此参数，会排除vendor.js三方文件库，
                // 如果设置为空数组则所有的文件都进行加固，并且不会排除vendor.js文件，需要手动配置
                // 数组中的路径必须为相对路径，并且不以 / 开头，相对于打包dist目录的路径            
                excludes:[]
        	}
        ]
    ]
});

# 3.加固说明

​ 1.压缩代码

​ 2.是否启用控制流扁平化(降低1.5倍的运行速度)

​ 3.应用概率;在较大的代码库中，建议降低此值，因为大量的控制流转换可能会增加代码的大小并降低代码的速度。

​ 4.随机的死代码块(增加了混淆代码的大小)

​ 5.死代码块的影响概率

​ 6.几乎不可能使用开发者工具的控制台选项卡

​ 7.会在“控制台”选项卡上使用间隔强制调试模式，从而更难使用“开发人员工具”的其他功能。

​ 8.通过用空函数替换它们来禁用console.log，console.info，console.error和console.warn。这使得调试器的使用更加困难。

​ 9.标识符的混淆方式 hexadecimal(十六进制) mangled(短标识符)

​ 10.是否启用全局变量和函数名称的混淆

​ 11.通过固定和随机（在代码混淆时生成）的位置移动数组。这使得将删除的字符串的顺序与其原始位置相匹配变得更加困难。

​ 12.混淆后的代码,不能使用代码美化

​ 13.删除字符串文字并将它们放在一个特殊的数组中

​ 14.允许启用/禁用字符串转换为unicode转义序列。Unicode转义序列大大增加了代码大小，并且可以轻松地将字符串恢复为原始视图。建议仅对小型源代码启用此选项。
