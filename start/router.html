# 路由

路由功能使用vue-router (opens new window)进行封装管理

文件地址：ynet 平台项目 ynet-core/src/core/router.ts

# API

属性 值 说明
routerDir prev | next 根据此值确定路由方向
方法 参数 返回值
push 参考vue-router push (opens new window)方法 Promise
replace 参考vue-router replace (opens new window)方法 Promise
back 无 无
配置项 参数
onBefore to, from, next（参照vue-router beforeEach (opens new window)方法）
onAfter 无，（参照 router.afterEach (opens new window)方法）
onExit 无，触发情况查看下方示例
disableRouterParamSession 禁用路由跳转时 param 存放 session，默认启用

# disableRouterParamSession

禁用路由跳转时 param 存放 session，默认启用

部分场景下在跨页面传值时，需要传递对象，并且不想把参数对外保护在 url 时，可以使用 params 方式进行传递

但是在使用 params 传递时，当页面刷新或者路由返回时，params 会丢失，造成页面数据无法展示，为了解决该问题，我们在平台默认提供了路由参数会话缓存存放的方案来解决该问题

默认所有的路由跳转通过 params 传值时，都会存放在会话缓存中，如果想禁用可以设置该参数为true

# 使用方式

  1. 传值时和 vue-router 使用方式一样

    this.$app.router.push({ name: "xxx", params: {} });

  2. 在取值时我们提供了两种方式

    1> 我们仍然保留了 vue-router 原始的取值方式，但是该方式去值会存在刷新页面/返回时参数丢失问题

    this.$rout.params;

    2> 我们提供了 getRouterParam 方法来进行参数取值，如果 params 有值则直接返回 route.params，如果没有则取会话缓存数据

    this.getRouterParam();

提示，我们内部对其进行了深度优化，当页面销毁时，我们会判断路由方向，只有在返回时，我们才会对该页面的参数缓存进行清除，正序操作为保证返回的业务路径完整性，不会清除会话缓存，所以对于页面中的 params 会话缓存的清空无需过度关心

针对上述逻辑，如果在返回页面时，跨级跳转跳过的中间页面所产生的会话数据不会被清除，如果想清楚可以使用 app.router.removeRouterParamFromSession 方法主动清除
